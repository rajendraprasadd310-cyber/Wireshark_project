# 🌐 Network Traffic Analysis Labs using Wireshark

## Overview

This repository contains hands-on network traffic analysis labs performed using **Wireshark** on **Kali Linux**. Each lab focuses on a different network protocol and demonstrates how to capture, inspect, and analyze packets for troubleshooting, network monitoring, and cybersecurity purposes.

These labs helped strengthen my understanding of network communication, protocol behavior, and packet analysis techniques commonly used by SOC Analysts and Network Security professionals.

---

## Tools Used

- Kali Linux
- Wireshark
- VirtualBox
- DHCP Enabled Network
- Internet Connectivity

---

## Labs Included

| Lab | Description |
|-----|-------------|
| **ARP Packet Analysis** | Captured ARP packets to observe IP-to-MAC address resolution and understand address mapping within a local network. |
| **ICMP Packet Analysis** | Analyzed ICMP packets generated by the `ping` command to study request and reply messages used for network troubleshooting. |
| **DHCP Packet Analysis** | Captured the complete DHCP DORA process (Discover, Offer, Request, ACK) to identify IP address assignment and lease information. |
| **SMTP Traffic Analysis** | Examined SMTP traffic to understand email communication and identify sender and receiver information. |
| **FTP Traffic Analysis** | Captured FTP traffic to observe file transfers and analyze transmitted filenames and commands. |
| **DNS Traffic Analysis** | Analyzed DNS queries and responses to understand domain name resolution and IP address mapping. |
| **HTTP Traffic Analysis** | Captured HTTP packets to inspect web requests, responses, and visited URLs. |
